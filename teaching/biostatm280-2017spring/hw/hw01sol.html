In [18]:
function g(k)
    for i = 1:10
        k = 5k - 1
    end
    k
end
Out[18]:
g (generic function with 1 method)

Computer integer arithmetic obeys the usual associative and distributive rules. LLVM is able to optimize, according to the associative rule, the loop

for i = 1:10
    k = 5k - 1
end

to

k = 9565625k - 2441406
In [19]:
code_llvm(g, (Int,))

define i64 @julia_g_71651(i64) #0 {
top:
  %1 = mul i64 %0, 9765625
  %2 = add i64 %1, -2441406
  ret i64 %2
}

Floating-point numbers do not necessarily obey the associative rule. This prevents LLVM from the simplification.

The @fastmath macro enables compiler optimization by assuming the associative law.

In [21]:
function g_fastmath(k)  
    @fastmath for i = 1:10  
        k = 5k - 1
    end
    k
end

  ret double %2
}
